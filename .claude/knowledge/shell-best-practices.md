# Shell Script Best Practices

vibemux プロジェクトで遵守するシェルスクリプトのベストプラクティス。

## 必須パターン

- `set -euo pipefail` を全スクリプト先頭に記述
- `local` で関数内変数を宣言
- 変数展開は必ずダブルクォート: `"$var"`
- コマンド置換は `$()` を使用（バッククォート禁止）
- `[[ ]]` を使用（`[ ]` は使わない）

## エラーハンドリング

- `|| true` で意図的にエラーを無視する場合はコメントで理由を明記
- `trap` でクリーンアップ処理を登録
- `die()` ヘルパーで統一的なエラー終了

## 移植性

- Bash 4.0+ を前提とする（macOS 標準の bash 3.x は対象外）
- GNU/BSD 両方で動く構文を優先（`sed -i` の差異に注意）
- `shellcheck` を通過しない構文は使用禁止

## テスト

- 外部コマンドのモック化より、実際のコマンド出力に対するテストを優先
- テストは `set -uo pipefail` で実行（`-e` はアサーション失敗の扱いに注意）
- 一時ファイルは `mktemp` で作成し `trap` で削除

## tmux 固有

- セッション名はユーザー入力 — 必ずクォートして使用
- `tmux has-session` で存在チェックしてからアタッチ
- ペインインデックスはハードコードせず変数管理
