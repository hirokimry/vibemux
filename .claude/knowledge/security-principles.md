# Security Principles

vibemux のセキュリティ判断基準。

## 脅威モデル

vibemux は tmux セッションランチャーであり、ユーザーのシェル環境で動作する。

### 保護すべき資産

1. **ユーザーの個人情報** — メールアドレス、名前がgit履歴やコードに漏洩しない
2. **シークレット** — APIキー、トークン、パスワードがリポジトリに混入しない
3. **シェル環境の安全性** — vibemux 経由でコマンドインジェクションが発生しない

### 攻撃面

- **セッション名**: ユーザー入力がそのまま tmux コマンドの引数になる
- **設定ファイル**: `source` で読み込むため、任意コード実行の可能性
- **環境変数**: `VIBEMUX_PANE_*` の値が `tmux send-keys` に渡る

## 対策

### 3層防御

1. **git hooks (pre-commit/pre-push)**: シークレット・メール・.env・大容量ファイル検出
2. **GitHub Actions CI**: 全PRに対するシークレットスキャン・shellcheck
3. **branch protection**: main への直接push禁止、CI必須

### コーディング規約

- 変数展開は必ずダブルクォートで囲む
- `eval` は使用禁止
- 外部入力（セッション名、ディレクトリパス）は検証してから使用
- 一時ファイルは `/tmp/.vibemux-*` プレフィックスで `mktemp` 使用

## 判断基準

- **シークレット検出**: 即座にブロック。疑わしきもブロック
- **個人情報**: git user.email は `noreply` 必須。コード内メールアドレスは警告
- **コマンドインジェクション**: 外部入力が制御構造に到達する経路は全て排除
